Q1: What is the purpose of the 'let' keyword in JavaScript?

Answer: The 'let' keyword is used to declare variables that are limited to the scope of a block statement, such as inside a loop or an if statement. It is preferred over 'var' due to its block-scope functionality and prevention of variable hoisting.

Q2: What is a closure in JavaScript?

Answer: A closure is a feature in JavaScript where an inner function has access to the outer (enclosing) function's variables. It allows for the creation of private variables and functions, providing a way to encapsulate state and behavior.

Q3: How does the 'this' keyword work in JavaScript?

Answer: The 'this' keyword in JavaScript refers to the object from which a function is called. Its value is determined by how a function is called (runtime binding). It can refer to different objects depending on the context, such as the global object, a specific instance, or a newly created object in a constructor function.

Q4: What is the difference between '==' and '===' in JavaScript?

Answer: The '==' operator checks for equality with type conversion, meaning it converts the operands to the same type before making the comparison. The '===' operator checks for strict equality, meaning it considers both the value and the type without performing type conversion. '===' is generally preferred to avoid unexpected type coercion.

Q5: What is an IIFE in JavaScript and why is it used?

Answer: An IIFE (Immediately Invoked Function Expression) is a function that is executed immediately after it is defined. It is used to create a private scope, avoiding variable collisions and keeping the global scope clean. It can also be useful for encapsulating logic.

Q6: What is event delegation in JavaScript?

Answer: Event delegation is a technique in JavaScript where a single event listener is added to a parent element to manage events for its child elements. It takes advantage of event bubbling, making it efficient by reducing the number of event listeners needed and dynamically handling events for future child elements.

Q7: How do you create a promise in JavaScript?

Answer: A promise in JavaScript can be created using the Promise constructor, which takes a function with two parameters: resolve and reject. These parameters are used to handle the successful or unsuccessful completion of an asynchronous operation. Example: new Promise((resolve, reject) => { /* async code */ });.

Q8: What is the purpose of async/await in JavaScript?

Answer: The async/await syntax in JavaScript is used to work with promises more comfortably. An async function returns a promise, and the await keyword is used to pause the execution of the function until the promise is resolved, making asynchronous code look and behave more like synchronous code, improving readability and maintainability.
